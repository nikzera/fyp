# Smart Prediction：自动化时期匹配与模型选择系统

> **模块**: `SpecialPeriod.py` — Smart Predict 模式（模式 2）  
> **目的**: 给定任意日期范围，自动识别市场状态、选择最优模型组合，并生成预测结果。

---

## 1. 设计动机

在 `SpecialPeriod.py` 的经典模式中，用户需要手动选择模型、市场和时期，这要求用户具备丰富的领域知识来判断哪些模型在何种市场条件下表现最优。**Smart Prediction** 将这一过程完全自动化：

- 用户只需输入**起始日期**、**结束日期**和**市场**
- 系统自动判断该时期属于哪种市场状态
- 根据状态类型和预测周期，自动选择并加权最优模型
- 自动完成训练、预测和结果输出

---

## 2. 系统架构

```
用户输入: 起始日期, 结束日期, 市场
                    |
    [步骤 1] 特征提取
        从实际市场数据中提取 6 维统计特征向量
                    |
    [步骤 2] 时期匹配
        与 7 个预定义 Special Period 进行
        z-score 标准化欧式距离比较
                    |
    [步骤 3] 类别映射
        将匹配到的时期 -> 4 种市场状态类别之一
                    |
    [步骤 4] 模型选择（按预测周期分别选择）
        使用市场特定的、周期感知的规则
        选择模型并分配集成权重
                    |
    [步骤 5] 训练与预测
        Walk-forward 训练 -> 加权集成预测
                    |
    输出: 预测结果、曲线图、SmartPredict_Summary.csv
```

---

## 3. 特征提取

对于任意日期范围，系统从实际市场价格数据中提取 **6 维特征向量**：

| # | 特征名称 | 计算公式 | 作用 |
|---|---------|---------|------|
| 1 | **均值收益率** (Mean Return) | `mean(对数收益率)` | 区分牛市/熊市/震荡市 |
| 2 | **波动率** (Volatility) | `std(对数收益率)` | 区分高波动危机与平稳趋势 |
| 3 | **最大回撤** (Max Drawdown) | `max(累计收益的running_max - 累计收益)` | 衡量下跌严重程度 |
| 4 | **偏度** (Skewness) | `skew(对数收益率)` | 左偏=急跌频繁；右偏=急涨频繁 |
| 5 | **峰度** (Kurtosis) | `kurtosis(对数收益率)` | 高峰度=肥尾/极端事件频繁（黑天鹅） |
| 6 | **趋势强度** (Trend Strength) | `sum(对数收益率) / 波动率` | 类似夏普比率；衡量趋势方向性 |

特征从 **Log Return 列**（缩放数据矩阵的第一列）计算，与训练管线保持一致。

## 4. 时期匹配算法

### 4.1 参考时期

系统维护 7 个预定义特殊时期作为参考基准：

| 时期 | 日期范围 | 核心特征 |
|------|---------|---------|
| 2008 金融危机 | 2007-10 ~ 2009-03 | 系统性崩盘，极高波动，极深回撤 |
| 2010 欧债危机 | 2010-04 ~ 2012-06 | 持续温和下跌，反复干预 |
| 2015 中国股灾 | 2015-06 ~ 2016-02 | 急跌后快速 V 型反弹 |
| 2018 贸易战 | 2018-01 ~ 2018-12 | 政策驱动，震荡，外生冲击 |
| 2020 COVID | 2020-02 ~ 2020-06 | 黑天鹅：史上最快崩盘 + 最快恢复 |
| 2022 加息周期 | 2022-01 ~ 2022-12 | 美联储紧缩，持续熊市 |
| 2023 AI 浪潮 | 2023-01 ~ 2023-12 | 科技驱动的持续牛市 |

### 4.2 距离计算

1. 计算用户输入时期的 6 维特征向量
2. 计算 7 个参考时期各自的 6 维特征向量
3. 将全部 8 个向量（7 个参考 + 1 个输入）堆叠，进行**联合 z-score 标准化**（每个特征维度零均值、单位方差）
4. 计算标准化后的输入向量与每个参考向量之间的**欧式距离**
5. 按距离排序 — 距离最近的参考时期即为最佳匹配

联合标准化确保所有特征无论原始尺度如何都有相等的贡献。

### 4.3 类别映射

每个参考时期映射到 **4 种市场状态类别** 之一：

| 类别 | 对应时期 | 特征描述 |
|------|---------|---------|
| `high_vol_crash` 高波动崩盘 | 2008 金融危机、2020 COVID | 极端波动、深度回撤、系统性风险 |
| `moderate_decline` 温和下跌 | 2010 欧债、2018 贸易战、2022 加息 | 渐进下跌、反复震荡、政策敏感 |
| `v_shape_recovery` V型反转 | 2015 中国股灾 | 急跌后快速反弹，深回撤但净趋势接近零 |
| `trending_bull` 趋势性牛市 | 2023 AI 浪潮 | 持续上涨、中等波动、强正向趋势 |

## 5. 模型选择策略

### 5.1 双层策略

系统采用**两级策略**，自动回退：

**策略 A — 历史数据驱动**（优先使用）：  
如果存在之前实验生成的 `SpecialPeriod_Summary.csv`，系统将：
1. 查找匹配时期在该 CSV 中各模型的实际 R² 分数
2. 按当前预测周期（horizon）筛选
3. 排除 ARIMA（基准）、SNN（不稳定）、PatchTST（除非长期趋势）
4. 选择 R² > 0 的前 K 个模型（默认 K=3）
5. 按 R² 比例分配权重（PatchTST 额外乘以 0.3 惩罚系数）

**策略 B — 启发式规则**（回退方案）：  
如果没有历史数据，系统使用从 SP500 和 HSI 综合实验中提炼的专家规则。

### 5.2 预测周期感知的模型选择

不同的预测周期需要不同的模型架构。系统将 horizon 分为三档：

| 档位 | Horizon 范围 | 设计理由 |
|------|:----------:|---------|
| **短期** (Short) | H ≤ 1 | 次日预测；近期模式主导 |
| **中期** (Medium) | H = 2~5 | 短期趋势跟踪；需要平衡的复杂度 |
| **长期** (Long) | H ≥ 6 | 多日前瞻；需更强泛化能力 |

**每个档位有独立的模型选择和权重分配**。一次 Smart Predict 运行若同时包含 H=1 和 H=10，会为两者使用**不同的模型组合**。

### 5.3 市场特化规则

SP500 和 HSI 的预测特性存在根本性差异。系统为每个市场维护**独立的规则集**：

#### SP500 规则

基于 SP500 实验分析（Seed2620）：
- **DL 模型排名**: Mamba (0.64) > Bi-RNN (0.63) > Bi-GRU (0.58) > Bi-LSTM (0.48)
- 模型间性能差距大 — 权重差异化很重要
- PatchTST 灾难性表现 (R² = -0.80)，始终排除
- SNN 极不稳定 (R² = -0.13)，始终排除

| 类别 | 短期 (H≤1) | 中期 (H=2~5) | 长期 (H≥6) |
|------|-----------|-------------|------------|
| `high_vol_crash` | Mamba 0.30, Bi-GRU 0.25, Bi-RNN 0.25, Bi-LSTM 0.20 | Bi-RNN 0.30, Mamba 0.25, Bi-LSTM 0.25, Bi-GRU 0.20 | Mamba 0.40, Bi-RNN 0.30, Bi-GRU 0.30 |
| `moderate_decline` | Mamba 0.30, Bi-GRU 0.25, Bi-RNN 0.25, Bi-LSTM 0.20 | Mamba 0.30, Bi-RNN 0.25, Bi-LSTM 0.25, Bi-GRU 0.20 | Mamba 0.40, Bi-RNN 0.30, Bi-GRU 0.30 |
| `v_shape_recovery` | Mamba 0.35, Bi-GRU 0.25, Bi-RNN 0.20, Bi-LSTM 0.20 | Bi-GRU 0.30, Bi-LSTM 0.25, Mamba 0.25, Bi-RNN 0.20 | Mamba 0.40, Bi-RNN 0.30, Bi-GRU 0.30 |
| `trending_bull` | Mamba 0.40, Bi-GRU 0.35, Bi-RNN 0.25 | Bi-GRU 0.35, Mamba 0.35, Bi-RNN 0.30 | Mamba 0.40, Bi-GRU 0.30, Bi-RNN 0.30 |

**SP500 核心规律**：
- Mamba 主导短期预测（H=1 时 4/7 个时期最优）
- Mamba 同样主导长期预测（所有类别 H≥6 权重最高）
- Bi-GRU 在趋势性和反转市场中表现突出

#### HSI 规则

基于 HSI 实验分析（Seed2620）：
- **DL 模型排名**: Bi-GRU (0.81) > Bi-RNN (0.81) > Mamba (0.80) > Bi-LSTM (0.80)
- Top-4 模型极为接近（仅 0.01 R² 差距）— 权重更均匀
- PatchTST 为正 (R² = 0.52)，但仍远低于 Top-4
- SNN 为正 (R² = 0.36)，但波动太大

| 类别 | 短期 (H≤1) | 中期 (H=2~5) | 长期 (H≥6) |
|------|-----------|-------------|------------|
| `high_vol_crash` | Bi-RNN 0.30, Bi-GRU 0.25, Bi-LSTM 0.25, Mamba 0.20 | Bi-RNN 0.30, Mamba 0.25, Bi-LSTM 0.25, Bi-GRU 0.20 | Bi-LSTM 0.40, Bi-RNN 0.30, Bi-GRU 0.30 |
| `moderate_decline` | Bi-RNN 0.30, Bi-GRU 0.30, Mamba 0.20, Bi-LSTM 0.20 | Bi-RNN 0.30, Bi-GRU 0.28, Mamba 0.22, Bi-LSTM 0.20 | Bi-GRU 0.35, Bi-RNN 0.35, Mamba 0.30 |
| `v_shape_recovery` | Bi-GRU 0.30, Mamba 0.25, Bi-LSTM 0.25, Bi-RNN 0.20 | Bi-LSTM 0.30, Bi-GRU 0.25, Mamba 0.25, Bi-RNN 0.20 | Bi-GRU 0.40, Bi-RNN 0.30, Bi-LSTM 0.30 |
| `trending_bull` | Mamba 0.30, Bi-GRU 0.25, Bi-LSTM 0.25, Bi-RNN 0.20 | Bi-GRU 0.28, Mamba 0.28, Bi-LSTM 0.22, Bi-RNN 0.22 | Mamba 0.35, Bi-GRU 0.35, Bi-RNN 0.30 |

**HSI 核心规律**：
- Bi-RNN 和 Bi-GRU 主导短期预测（H=1 时 5/7 个时期最优）— 与 SP500 相反
- Bi-LSTM 在 HSI 上远强于 SP500（R² 提升 +0.32）
- Mamba 仅在 `trending_bull` 类别中领先 — 不像 SP500 那样全面主导
- 权重更均匀，因为 Top-4 模型性能极为接近

### 5.4 PatchTST 加入条件

PatchTST **默认排除**（SP500 上 R² = -0.80）。但在以下条件同时满足时**重新加入**：

1. 输入时期跨度 **> 3 年**
2. 匹配到的类别为**趋势性**类型（`trending_bull`、`high_vol_crash` 或 `moderate_decline`）

加入时，PatchTST 获得固定权重 **0.20**，其他模型权重重新归一化。

**原因**: PatchTST 的 patch 注意力机制在持续方向性行情中能捕捉长程模式。在 HSI 上它达到正 R²（0.52），在 SP500 上也在 2023 AI 浪潮（强趋势市场）中显示出边际正 R²。但对于短期或高波动时期，其时间分辨率的损失是致命的。

### 5.5 永久排除的模型

| 模型 | 排除原因 |
|------|---------|
| **ARIMA** | 仅作为统计基准参照，不参与集成预测 |
| **SNN** | 极度不稳定（SP500 R² 范围：-1.51 至 0.56；HSI 标准差 = 0.33），不可靠 |

## 6. 集成方法

### 6.1 加权平均集成

最终预测为各模型预测的**加权平均**：

$$\hat{y}_{ensemble} = \frac{\sum_{i=1}^{K} w_i \cdot \hat{y}_i}{\sum_{i=1}^{K} w_i}$$

其中：
- $w_i$ = 分配给模型 $i$ 的权重（来自启发式规则或历史 R²）
- $\hat{y}_i$ = 模型 $i$ 的预测值
- $K$ = 选中的模型数量（通常 3~4 个）

### 6.2 为什么加权优于等权

经典模式的原始 Ensemble 对所有模型使用等权平均，当弱模型（PatchTST、SNN）被包含时**会拖低整体性能**。SP500 实验显示 Ensemble（R² = 0.57）不如 Mamba（0.64）和 Bi-RNN（0.63），正是因为弱模型的污染。

Smart Prediction 通过以下方式解决：
1. **排除**已知的弱模型
2. **按预期性能比例加权**各模型
3. **按预测周期和市场自适应**调整权重

## 7. Walk-Forward 训练协议

Smart Prediction 使用与经典模式相同的 Walk-Forward 方法论：

```
|<------- 训练数据 ------->|<-- 用户时期 -->|
   时期开始前的所有数据         用户输入的时期

训练集:   [数据起点, 时期开始)
验证集:   训练集最后 10%（用于早停）
测试集:   时期数据（带 seq_len 前序回看）
```

- 模型**仅在时期开始前的数据上训练** — 无未来数据泄露
- 早停机制（patience=15）防止过拟合
- 每个（窗口大小, 预测周期）组合独立训练全新模型

## 8. 使用方法

### 8.1 运行 Smart Predict

```bash
python SpecialPeriod.py
```

在提示中选择**模式 2**：

```
========== Mode Selection ==========
  1. Classic Mode (manual selection of models/periods)
  2. Smart Predict (auto-match period & select best models)
====================================
Enter mode (1 or 2): 2
```

然后依次输入：
1. 市场（SP500 或 HSI）
2. 起始日期（YYYY-MM-DD）
3. 结束日期（YYYY-MM-DD）
4. 序列长度（窗口大小）
5. 预测周期

### 8.2 输出内容

系统输出：
- **特征向量**: 输入时期的 6 维统计特征
- **相似度排名**: 与全部 7 个参考时期的距离
- **最佳匹配**: 最近的时期及其所属类别
- **模型选择**: 每个 horizon 的模型列表、权重及使用的策略
- **预测结果**: 每个模型和加权集成的 MAE、R²、RMSE
- **预测曲线图**: 每个模型和集成的实际值 vs 预测值图
- **汇总 CSV**: `SmartPredict_Summary.csv`

## 9. 设计决策与理由

| 决策 | 理由 |
|------|------|
| 使用 6 维特征而非原始价格 | 对价格水平不变性；捕捉市场状态特征（波动率、趋势、尾部风险） |
| 联合 z-score 标准化 | 确保所有特征对距离有相等贡献；防止波动率维度主导 |
| 欧式距离而非余弦相似度 | 我们关心幅度差异（2 倍波动率的时期确实不同），而非方向相似性 |
| 4 个类别而非 7 个 | 相似时期共享最优模型配置；分组减少对特定历史事件的过拟合 |
| 按 Horizon 分别选模型 | SP500 分析表明 R² 从 H=1 到 H=10 下降 83%，且不同 horizon 的最优模型完全不同 |
| SP500/HSI 分离规则 | HSI 的 Top-4 DL 模型仅有 0.01 R² 差距，而 SP500 有 0.16 差距 — 需要根本不同的加权策略 |
| PatchTST >3 年门槛 | PatchTST 需要长程模式才能发挥作用；短期波动时期会摧毁其 patch 注意力机制 |
| 加权集成优于等权集成 | 防止弱模型拖累整体；经典模式的 Ensemble 因 PatchTST/SNN 污染而表现不如单模型 |
